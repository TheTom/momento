CLAUDE.md Trigger Rules

Cadence-based — not threshold-based. Model can't measure context %.
after subtask → log_knowledge(session_state)
after /clear → retrieve_context()
session start → retrieve_context()
"checkpoint" → log_knowledge(session_state)
before /compact → log_knowledge(session_state)
unfamiliar error → retrieve_context(query=...)
decision finalized → log_knowledge(decision/plan)
Agent Adapter Layer — Same core, different trigger wiring
Momento's MCP surface is agent-agnostic. Adapters are thin instruction blocks that tell each agent when to call tools — not how they behave. No code changes. No schema changes. No new tools.

Restore Mode — 5-Tier Ordering

Surface+branch-aware: prefers entries matching cwd surface and current git branch. Preference, not filter. Surface > branch in sort priority.
1 session_state — surface-matching, up to 4
1 session_state — other surfaces, up to 2
2 plan — finalized plans, up to 2
3 decision — high confidence, up to 3
4 gotcha + pattern — up to 4 combined
5 cross-project — matching tags, up to 2
