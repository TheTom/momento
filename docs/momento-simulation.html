You switch agents mid-work.
What happens to your context?

A realistic walkthrough. Same project, two different AI coding agents, one mid-day switch. No idealized demo — just a normal Tuesday.

The Project

SaaS app. Node backend, Swift iOS client. Stripe billing, server-side auth. The kind of codebase where decisions made on Tuesday affect Wednesday's debugging.

You start the day in Claude Code. After lunch, you'll switch to Codex for a different iteration style. Same repo. Different brain.

10:00 AM — Claude Code
Backend subscription upgrades

You're implementing the upgrade flow. Endpoint, Stripe integration, webhook handling. The kind of work where each decision constrains the next one.

  • Created POST /api/upgrade endpoint
  • Stripe checkout session created server-side
  • Idempotent upgrade state check in SubscriptionService
  • Fixed race condition — webhook firing before client state refresh

As you work, Claude checkpoints at natural boundaries. You don't ask for this — the agent is instructed to save after completing significant work. It happens quietly.

Two entries get saved to Momento:

Saved automatically session_state
Upgrade endpoint implemented. Stripe checkout session server-side. Webhook race condition fixed by persisting state before event emit. Next: reconcile subscription status on client refresh.
Saved automatically decision
Decision: Server-side Stripe checkout
Rationale: Avoid exposing secret keys in client
Rejected: Client-generated checkout
Implications: Webhook reconciliation required

You keep working. Everything feels normal. You don't think about memory. That's the point.

11:30 AM — You switch agents

You close Claude Code. Open Codex in the same repository. Brand new session. Codex has zero memory of what happened this morning.

11:30 AM — Codex session starts
Momento restores context

Before you even finish typing your first prompt, the new agent has context.

When the Codex session starts, it automatically retrieves context for the current project. Momento returns what matters — structured, concise, under a second. It does not replay the conversation. It restores the distilled state of the project:

Why this matters

Different agents have different strengths. You might like Claude for architectural reasoning. You might prefer Codex for tight refactors. You might use something else tomorrow.

Momento makes the agent interchangeable. Your project memory stays stable. Your tool can change. The memory layer stays constant. The reasoning engine can change.

Bonus: Two Branches, One Repo
What about parallel work?

Same project. Two branches. Radically different context.

10:30 AM on feature/billing-rewrite: Replacing JWT with session tokens entirely. Different architecture. Save a decision about the new auth model.

Memory is ranked, not partitioned.
